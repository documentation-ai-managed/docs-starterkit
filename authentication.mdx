## Overview

Secure your StarterKit projects with Documentation.AI by using API keys or OAuth. API keys suit server-side integrations, while OAuth enables user-based access. Permissions follow role-based access control (RBAC) to restrict actions like read, write, or admin.

<Callout kind="info">

Before starting, create a Documentation.AI account at `https://dashboard.example.com` and generate credentials.

</Callout>

## Authentication Methods

Choose the method that fits your use case.

<Tabs>
  <Tab title="API Keys" icon="key">

    Generate API keys from your dashboard.

    <Steps>
      <Step title="Create Key" icon="plus">
        Navigate to `https://dashboard.example.com/account/api-keys`.
        
        Click **Generate New Key**.
      </Step>
      <Step title="Copy Key" icon="copy">
        Copy the generated key securely.
      </Step>
      <Step title="Use in StarterKit" icon="code">
        Set the key as an environment variable.
      </Step>
    </Steps>

    <CodeGroup tabs="Node.js,Python">
      ````javascript
      import { StarterKit } from '@documentation-ai/starterkit';

      const client = new StarterKit({
        apiKey: process.env.STARTERKIT_API_KEY
      });

      const docs = await client.docs.list();
      console.log(docs);
      ````

      ````python
      from starterkit import StarterKit

      client = StarterKit(api_key="YOUR_API_KEY")

      docs = client.docs.list()
      print(docs)
      ````
    </CodeGroup>

    <ParamField header="Authorization" param-type="string" required="true">
      Bearer token format: `Bearer YOUR_API_KEY`.
    </ParamField>

  </Tab>
  <Tab title="OAuth 2.0" icon="shield">

    Implement OAuth for client-side apps.

    <Steps>
      <Step title="Register App" icon="settings">
        In dashboard, register your app at `https://dashboard.example.com/apps`.
        
        Note the `client_id` and `client_secret`.
      </Step>
      <Step title="Redirect User" icon="external-link">
        Direct users to `https://auth.example.com/oauth/authorize?client_id=YOUR_CLIENT_ID&redirect_uri=YOUR_URI&response_type=code`.
      </Step>
      <Step title="Exchange Code" icon="arrow-right">
        Exchange authorization code for tokens.
      </Step>
    </Steps>

    <CodeGroup tabs="Node.js,cURL">
      ````javascript
      const response = await fetch('https://auth.example.com/oauth/token', {
        method: 'POST',
        headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
        body: new URLSearchParams({
          grant_type: 'authorization_code',
          client_id: 'YOUR_CLIENT_ID',
          client_secret: 'YOUR_CLIENT_SECRET',
          code: 'AUTH_CODE',
          redirect_uri: 'YOUR_URI'
        })
      });

      const { access_token } = await response.json();
      ````

      ````bash
      curl -X POST https://auth.example.com/oauth/token \
        -H "Content-Type: application/x-www-form-urlencoded" \
        -d "grant_type=authorization_code&client_id=YOUR_CLIENT_ID&client_secret=YOUR_CLIENT_SECRET&code=AUTH_CODE&redirect_uri=YOUR_URI"
      ````
    </CodeGroup>

  </Tab>
</Tabs>

## Role-Based Permissions

StarterKit uses RBAC with these predefined roles:

| Role     | Read | Write | Admin | Description                  |
|----------|------|-------|-------|------------------------------|
| Viewer   | ✅   | ❌    | ❌    | Read documents only         |
| Editor   | ✅   | ✅    | ❌    | Edit and read documents     |
| Admin    | ✅   | ✅    | ✅    | Full access, manage users   |

Assign roles via dashboard or API:

<ParamField path="userId" param-type="string" required="true">
  Target user ID.
</ParamField>

<ParamField body="role" param-type="string" required="true">
  Role name: `viewer`, `editor`, or `admin`.
</ParamField>

## Security Best Practices

<Callout kind="alert">

Never expose API keys in client-side code. Rotate keys regularly and use least-privilege roles. Monitor usage at `https://dashboard.example.com/usage`.

</Callout>

## Troubleshooting

<ExpandableGroup>
  <Expandable title="Invalid API Key" default-open="true">

    **Error:** `401 Unauthorized`

    Check:
    - Key copied correctly (no extra spaces).
    - Environment variable set: `echo $STARTERKIT_API_KEY`.
    - Key not revoked in dashboard.

    Test with cURL:

    ````bash
    curl -H "Authorization: Bearer YOUR_API_KEY" https://api.example.com/starterkit/docs
    ````

  </Expandable>
  <Expandable title="OAuth Redirect URI Mismatch">

    Ensure your `redirect_uri` matches exactly what's registered. Use `http://localhost:3000/callback` for development.

  </Expandable>
  <Expandable title="Permission Denied">

    Verify user role allows the action. Upgrade to `admin` for testing, then downgrade.

  </Expandable>
</ExpandableGroup>
