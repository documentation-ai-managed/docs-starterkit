## Recent Updates

Stay current with StarterKit's evolution. You receive new features, performance improvements, and critical fixes in each release. Check the tags on each update to prioritize your migration.

<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

## New Features

- Added support for custom brand colors via `BRAND_COLOR` environment variable. Set it to `#3B82F6` for Documentation.AI matching.
- Introduced `<Video>` and `<Iframe>` components for richer media embeds in your docs.

## Improvements

- Enhanced MDX parsing with better escaping for `{` and `}` characters in prose.
- Optimized build times by 30% with parallel component compilation.

## Bug Fixes

- Fixed nested code block rendering inside `<Steps>` and `<Tabs>`.
- Resolved heading hierarchy warnings in strict mode.

</Update>

<Update label="2024-09-20" description="v1.1.0" tags={["feature", "bugfix"]}>

## New Features

- New `<Columns>` component for responsive card grids (up to 5 columns).
- Integrated Lucide icons with auto-completion in your IDE.

## Bug Fixes

- Corrected attribute parsing for arrays like `tags={["feature"]}`.
- Patched self-closing component errors in JSX validation.

</CodeGroup>

</Update>

<Update label="2024-08-10" description="v1.0.0" tags={["breaking", "feature"]}>

## New Features

- Core MDX components: `<Callout>`, `<Steps>`, `<Tabs>`, and `<Update>`.
- Full parsing rules for escaping `<`, `>`, `{`, `}` in documentation.

## Breaking Changes

- Removed legacy `<Mermaid>` component syntax. Use code blocks only:
  ```mermaid
  graph TD
      A[User] --> B[Docs]
      B --> C[Parsed MDX]
  ```

## Migration Notes

Update your config to use double-quoted attributes everywhere.

</Update>

## Breaking Changes Summary

Review these changes before upgrading:

| Version | Change | Impact |
|---------|--------|--------|
| v1.2.0 | Custom brand color env var | Low - Optional |

</Callout>

## Migration Guide

Follow these steps to upgrade smoothly.

<Steps>
  <Step title="Backup" icon="archive">
    Save your current docs folder.

    ```
    cp -r docs docs-backup
    ```
  </Step>
  <Step title="Update Package" icon="package">
    Install the latest version.

    <CodeGroup tabs="npm,yarn">
    ````bash
    npm install starterkit@latest
    ````
    ````bash
    yarn add starterkit@latest
    ````
    </CodeGroup>
  </Step>
  <Step title="Review Config" icon="settings">
    Check `mdx.config.js` for deprecated props.

    ```javascript
    // Update to new format
    module.exports = {
      brandColor: process.env.BRAND_COLOR || '#3B82F6',
      escapeSpecial: true, // Now default
    };
    ```
  </Step>
  <Step title="Test Build" icon="check-circle">
    Run your build and scan for parsing errors.
  </Step>
</Steps>
