## 1. Update a page

<Steps>
  <Step title="Edit introduction.mdx" icon="edit">
    Replace the sample content with your product overview and first links.
  </Step>
  <Step title="Preview your change" icon="eye">
    Use your usual preview workflow (local dev server or hosted editor) to confirm the edit.
  </Step>
</Steps>

## 2. Add a new page

<Steps>
  <Step title="Create a new .mdx file" icon="file-text">
    Example starter frontmatter:

```mdx
---
title: Your First Page
description: Replace this with your own summary.
---

## Your First Page
Write the content you want users to read first.
```
  </Step>
  <Step title="Register it in documentation.json" icon="settings">
    Add a new page entry so it appears in navigation.
  </Step>
</Steps>

## 3. Connect API Reference (optional)

If you have an API, replace `openapi.yaml` with your real OpenAPI file. If you do not need an API reference, remove that tab from `documentation.json`.
