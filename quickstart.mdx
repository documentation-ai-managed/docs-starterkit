## Prerequisites

Before starting, ensure you have the following installed:

<Callout kind="info">

- Node.js version `>=18.0.0` (download from [nodejs.org](https://nodejs.org))
- npm, yarn, or pnpm package manager
- Git for version control (optional but recommended)

</Callout>

Verify your Node.js version by running `node --version` in your terminal. You should see output like `v20.10.0`.

## Install StarterKit

Use the create command to scaffold your first documentation project. StarterKit generates a complete MDX-based documentation site ready for customization.

<Steps>
  <Step title="Create New Project" icon="download">
    Run the creation command with your preferred package manager. Replace `my-docs` with your project name.

    <CodeGroup tabs="npm,yarn,pnpm">
      ````bash
      npm create starterkit@latest my-docs
      ````

      ````bash
      yarn create starterkit my-docs
      ````

      ````bash
      pnpm create starterkit my-docs
      ````
    </CodeGroup>

    Follow the interactive prompts to select features like dark mode or search integration.
  </Step>

  <Step title="Navigate to Project" icon="folder">
    Change into your new project directory:

    ````bash
    cd my-docs
    ````
  </Step>
</Steps>

## Run the Development Server

Start the local development server to preview your site.

<Tabs>
  <Tab title="npm" icon="package">
    ````bash
    npm install
    npm run dev
    ````
  </Tab>

  <Tab title="yarn" icon="package">
    ````bash
    yarn
    yarn dev
    ````
  </Tab>

  <Tab title="pnpm" icon="package">
    ````bash
    pnpm install
    pnpm dev
    ````
  </Tab>
</Tabs>

Your site runs at `http://localhost:3000`. Open this URL in your browser to see the default documentation homepage.

## Verify the Setup

Once the server starts, you should see sample pages like the introduction and changelog. Check the console output for confirmation:

````bash
✓ Server running at http://localhost:3000
✓ Documentation built successfully
✓ Hot reload enabled
````

Navigate to pages such as `/introduction` and `/changelog` to confirm navigation works. Edit `src/content/introduction.mdx` and save to test hot reloading – changes appear instantly.

<Callout kind="success">
  Success! Your StarterKit project is ready. You now have a fully functional documentation site.
</Callout>

## Customize Your Site

Edit the `config/site.ts` file to set your brand details:

````typescript
export default {
  title: 'Documentation.AI',
  brandColor: '#3B82F6',
  nav: [
    { title: 'Introduction', href: '/introduction' },
    { title: 'Quickstart', href: '/quickstart' },
  ],
};
````

<Expandable title="Troubleshooting Common Issues" default-open="false">

If the server fails to start:

- Clear node_modules: `rm -rf node_modules && npm install`
- Check port 3000: Use `npm run dev -- --port 3001`
- Ensure no typos in project name (use lowercase with hyphens)

For build issues, run `npm run build` and review errors.
